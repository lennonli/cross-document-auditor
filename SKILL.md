# 跨机构底稿智能交叉审计系统 (Cross-Document Auditor)

## 🎯 核心使用场景与触发条件
当用户指令涉及以下意图时，必须强制触发本技能工作流：
- “对比招股说明书与法律意见书的差异”
- “核对审计报告财务明细是否准确写入招股书”
- “找出多份底稿之间的数据打架或事实矛盾”

## ⚠️ 核心纪律（防呆原则）
本技能是对多次实战复盘后的终极解决方案，请在执行时严格遵循：
1. **绝对禁止全文逐字对比**：禁止建议用户使用 Beyond Compare 等工具。不同机构行文习惯不同，全文比对会产生95%的无效噪音。
2. **强制双向集合核查（Bidirectional Check）**：清单类数据必须计算“A独有”和“B独有”的差集，单向检索必定导致漏报！
3. **禁止复杂表格扁平化搜索**：财务数据不能转为纯文本搜索，必须结构化提取。
4. **强制拦截重大性遗漏（Materiality）**：对“A有、B无”的信息不能机械忽略，必须常识判定其是否属于重大信披缺失（如资金占用、重大违规）。

## 🛠️ 标准操作程序 (SOP)

### 第一步：深度文档解析与痕迹抓取 (Deep Extraction)
1. **纯净版提取**：转换 docx 为 markdown。
2. **修订痕迹提取**：使用 `pandoc --track-changes=all` 抓取被“划掉/删除”的实体名称，寻找“删漏了”的靶标。
3. **表格结构化提取**：使用脚本剥离数据表为 DataFrame 数组，保留[实体, 年份, 科目]三维坐标。

### 第二步：单文件内部自洽性扫描 (Intra-Document Consistency)
在跨文件对比前，必须“除内鬼”：
1. **实体定性自洽**：如同文件前文写“当前控制”，后文写“已注销”，标为 P0 级矛盾。
2. **数字全局自洽**：同文件内同一指标（如员工人数 245 vs 252）不一致，立即标记。

### 第三步：静态比对——实体清单“双向差集” (Entity Set Math)
1. 提取 A 文件的实体全集：Set_A；提取 B 文件的实体全集：Set_B。
2. 输出 `Set_A - Set_B` 与 `Set_B - Set_A`。找出一篇底稿覆盖了、但另一篇遗漏核查的公司或专利。

### 第四步：静态比对——结构化 KV 属性对撞 (Attribute Collision)
建立“实体别名词典”（发那特=抚州发那特），严格校验对象属性：
- **比例卡位**：拦截 `100% vs 80%` 的持股认定差异。
- **金额卡位**：拦截 `500.00 vs 3,500.00` 的笔误。

## 🚀 进阶内核：动态逻辑验证 (Advanced Dynamic Validation v2.0)

### 第五步：跨文件“财务勾稽关系”自动验算 (Cross-Document Articulation)
不仅做相等判定（==），强制引入**代数求和判定（SUM）**：
- 当 A 文件呈现聚合总数，B 文件呈现离散明细时，系统必须自动加总 B 文件的明细数据（如：子公司1采购 + 子公司2采购 = 合并报表采购总额），再与 A 文件的总数进行核对闭环。

### 第六步：全量事件的“时间线向量校验” (Chronological Vectoring)
针对历史沿革、股权流转等高频出错章节：
- 对目标公司抽取所有带日期（年月日）的事件，生成时间轴向量序列。
- 对比 A 向量与 B 向量。如发现 `[2010代持 -> 2017解除]` vs `[2010代持 -> 2013增资 -> 2017解除]`，立即爆出“历史发展存在时间断层”预警。

### 第七步：责任主体口径的“降噪过滤” (Verification Boundary Filter)
在提取和比对比例、覆盖率等数据前，先提取数据的**主语/核查视角**：
- 若数据分别是“保荐机构核查覆盖 75%”与“本所律师核查覆盖 68%”，系统自动识别口径不同，将其从 P0 级矛盾降级为“合理差异”，实现智能降噪。

## 📊 输出报告分级规范
1. **P0 级 - 实质性矛盾与内伤**：（例：持股比例打架、同一文件内部自相矛盾、清单缺漏）。要求：立即修改。
2. **P1 级 - 重大信披遗漏**：（例：资金占用、重大违规未批，或时间线严重断层）。要求：强制增补。
3. **P2 级 - 口径与详略差异**：（例：券商与律师走访覆盖率不同）。要求：中介备好工作底稿即可。
4. **验证通过项**：列出100%匹配的核心财务与法律结论，提供审查背书。
