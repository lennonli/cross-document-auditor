# 跨机构底稿智能交叉审计系统 (Cross-Document Auditor)

## 🎯 核心使用场景与触发条件
当用户指令涉及以下意图时，必须强制触发本技能工作流：
- “对比招股说明书与法律意见书的差异”
- “核对审计报告财务明细是否准确写入招股书”
- “找出两份/多份底稿之间的数据打架或事实矛盾”
- “看 A 文件有没有覆盖 B 文件的内容”

## ⚠️ 核心纪律（防呆原则）
本技能是对多次实战（含大模型缺陷）复盘后的终极解决方案，请在执行时严格遵循：
1. **绝对禁止全文逐字对比**：禁止建议用户使用 Beyond Compare 等比对工具。券商、律所、会所的行文习惯完全不同，全文比对会产生95%的无效垃圾噪音。
2. **强制执行双向集合核查（Bidirectional Check）**：清单类数据（关联方、专利、注销企业等）必须计算“A独有”和“B独有”的差集，单向检索必定导致漏报！
3. **禁止将复杂表格扁平化搜索**：财务数据不能转为普通纯文本搜索，极易发生“行头/列头”错位。
4. **强制拦截重大性遗漏（Materiality）**：对于“A有、B无”的信息，不能机械性忽略。必须引入常识判断其是否影响信披完整性（如：大额资金占用、巨额诉讼、极端客户依赖等必须披露）。

## 🛠️ 标准操作程序 (SOP)

### 第一步：深度文档解析与痕迹抓取 (Deep Extraction)
不可仅提取最终净稿，必须同时捕捉隐藏信息：
1. **纯净版提取**：转换 docx 为 markdown，用于常规比对。
2. **修订痕迹提取**：使用 `pandoc --track-changes=all` 提取带有删除/新增标记的版本。
   * **目的**：重点抓取被“划掉/删除”的实体名称（如：某公司在最新版法律意见书中已被删除），为后续查找另一份文件是否“删漏了”提供靶标。
3. **结构化表格提取**：使用脚本（如 `pandas`、`python-docx`）将带数据表的部分专门剥离成 JSON 或 DataFrame 数组，保留[实体名称, 年份, 科目]三维坐标。

### 第二步：单文件内部自洽性扫描 (Intra-Document Consistency)
在跨文件对比前，必须先“除内鬼”：
1. **实体定性自洽**：搜索特定实体在同一文档所有出现位置的定性描述。如发现某主体在前文表述为“当前控制”，在后文表述为“已注销”，立即作为 **P0 级矛盾** 预警。
2. **数字全局自洽**：提取文中同一概念的数据（如“员工人数”、“总资产”），若同文件内出现 245 vs 252，立即标记。

### 第三步：实体清单的“双向差集”核对 (Entity Set Math)
针对所有列举型事实（如：董监高名单、关联方名单、对外投资、重大合同、专利明细）：
1. 提取 A 文件的实体全集：Set_A
2. 提取 B 文件的实体全集：Set_B
3. 输出 `Set_A - Set_B`：找出被 B 文件遗漏核查的实体。
4. 输出 `Set_B - Set_A`：找出被 A 文件遗漏核查的实体。
   * **价值**：瞬间发现券商披露了 20 家注销企业，而律师只核查了 18 家的严重合规漏洞。

### 第四步：结构化财务与股权数据对撞 (Structured Attribute Collision)
建立“实体别名词典”（如：`{发那特: 抚州发那特}`，应对不同中介简称差异）。
对提取出的对象属性进行严格的 KV (Key-Value) 值相等性校验：
- **持股比例**：拦截 `100% vs 80%` 这类致命的股权认定差异。
- **金额卡位**：按 [交易对手, 交易年度] 锁定金额。拦截 `500.00 vs 3,500.00` 的统计笔误。

### 第五步：重大性信披漏斗过滤 (Materiality Assessment)
当发现某些详细论述“在律师稿中有，在招股书中无”时：
1. **常规细节（过滤）**：如第三方客户销售价格对比、律师极度繁琐的核查程序说明。这些属于合理省略。
2. **重大风险点（报警）**：如高达数百万元的**历史资金占用**、高达 90% 以上的**单一关联方商标依赖**、隐瞒的**超产能具体基数**。此类缺失必须标记为 **P1 级重大信披缺失**。

---

## 📊 输出报告规范
最终输出给用户的比对报告必须严密分级：
1. **P0 级 - 实质性矛盾与内伤**：（例：持股比例打架、同一文件内部定性相左、关联方名录缺漏）。要求：立即修改。
2. **P1 级 - 重大信披遗漏**：（例：资金占用、重大违规罚款金额未披）。要求：强制增补以防问询。
3. **P2 级 - 口径与详略差异**：（例：金额包含/不包含税、核查时间段差异）。要求：券商备好底稿说明即可。
4. **验证通过项**：必须列示“已严格核对且数据100%一致”的核心条目清单，给用户提供审查安全感。
